Unari Sagi Operator 使い方マニュアル

このページは、アプリの導入、チェック用ログイン作成、詐欺チェック、シート反映までをメンバーが自分のMacで進めるための手順書です。

最新版DMGを開く 手順を見る 困った時

最初に知ること

このアプリは、Instagramアカウントに「詐欺表示」が出ているかをまとめて確認し、Google SheetsのD列へ結果を書き戻すためのものです。

メンバーがやること

アプリを入れる、初回セットアップを緑にする、チェック用ログインを作る、件数確認してから本番チェックを押す、最後にD列反映を確認する。

入力してはいけない場所

Instagramのパスワードや2FAコードは、Operator画面やログには入力しません。Android画面のInstagramアプリ内だけに手入力します。

1 DMGを入れる GitHub Releaseから最新版をダウンロードします。
2 初回セットアップ Python、Android、Instagram導入を緑にします。
3 ログイン追加 チェック用InstagramをAndroid側でログインします。
4 チェック実行 件数確認、本番チェック、D列反映の順に進めます。

用意するもの

必要なもの
  • Apple Silicon Mac
  • 安定したWi-Fi、またはiPhoneテザリング
  • チェック用Instagramアカウントのusername / password
  • 対象のGoogle Sheetsを開ける状態
入れなくてよいもの
  • Python
  • Homebrew
  • Java
  • Android Studio

必要なPython、Java、Android SDKはアプリ側のセットアップで用意します。

1. アプリを入れる

  1. 最新版Releaseを開き、Unari Sagi Operator.dmg をダウンロードします。
  2. ダウンロードしたDMGをダブルクリックします。
  3. 中にある Unari Sagi Operator.app を アプリケーション にドラッグします。
  4. アプリケーション から Unari Sagi Operator.app を右クリックし、開く を押します。
  5. macOSの確認画面が出たら、開く または 許可 を押します。
古いアプリがある場合

新しいDMGを入れたら、必ず アプリケーション の中のアプリを置き換えてから開き直します。DMGの中から直接起動しないでください。

2. 初回セットアップを緑にする

アプリが開くと、ブラウザにOperator画面が出ます。上部の 初回セットアップ を開き、赤い項目を上から順番に進めます。

  1. 迷ったら まとめて導入（①〜⑤） を押します。
  2. 個別に進める場合は ① Python環境 から順に押します。
  3. ② Android SDK は数GBのダウンロードがあるため、しばらく待ちます。
  4. ③ Android画面作成 を押し、Android画面の準備を待ちます。
  5. ④ 通信用設定 を押します。
  6. ⑤ Instagram導入 を押します。
  7. シートを使う前に ⑥ Google Sheets接続設定 を確認します。
  8. 最後に 状態更新 を押し、主要項目が緑になったことを確認します。
待つ目安

ログ欄が止まって見えても、ダウンロード中なら数分待ちます。ボタンを連打しません。

3. チェック用ログインを作る

  1. チェック用ログイン追加 を開きます。
  2. Instagram username を入れます。
  3. MacがiPhoneテザリングに接続済み にチェックします。
  4. チェック用ログインを1つ作る（推奨） を押します。
  5. Android画面のInstagramログイン画面で、username / password を手入力します。
  6. メール確認や2FAが出たら、録画や画面共有を止めてAndroid画面で完了します。
  7. フィードが表示されたら、そのまま待ちます。アプリが自動で必要な情報を取り込みます。
  8. 完了後、必要な場合だけ 作ったログインを確認 を押します。
重要

Operator画面にはpassword欄がありません。password、2FA、認証コードはAndroid側だけに入力します。

成功の目安
  • チェック用ログインが1つ増える
  • 作ったログインを確認 が成功する
  • 画面の「次の一手」が完了系になる

4. 詐欺チェックを実行する

  1. 詐欺チェック実行 を開きます。
  2. Google Sheets URL を貼ります。
  3. タブ名を入れます。
  4. ① まず件数を確認（本番はまだ走りません） を押します。
  5. 対象件数と必要なチェック用ログイン数を見ます。
  6. 問題なければ ② 本番チェックを実行 を押します。
  7. 確認画面で件数を見てからOKを押します。
  8. 完了後、結果CSV欄に logs/sagi_operator_result_...csv が入ったことを確認します。
  9. ③ 書き戻さずに件数だけ確認（安全） を押します。
  10. 件数が合っていれば ④ シートのD列に反映（確定） を押します。
初回検証だけの時

大量チェックしたくない場合は、①の件数確認で止めます。②本番チェックや④D列反映は押しません。

5. 完了確認

  • 対象シートのD列に 詐欺表示 / なし / アカウント不明 が入っている
  • 対象件数と、書き戻し件数が合っている
  • Slack通知が必要な運用では、通知が届いている
  • 途中で止まった場合は、結果CSVを消さずに残している
Slack通知だけ失敗した場合

シートD列が更新済みなら、チェック結果自体は残っています。Slack通知のエラーは管理者へ渡してください。

困った時

まず画面のログ末尾をコピーし、必要ならスクショと一緒に藤巻または管理者へ渡します。password、2FA、cookie、sessionidが見える画面は共有しません。

画面が開かない

~/Library/Logs/UnariSagiOperator/ の最新ログを渡します。

Android未接続 / AVD boot timeout

アプリを開き直し、Android準備だけやり直す を押します。繰り返す場合はMac再起動後に再実行します。

NEEDS_SUPPLEMENT

チェック用ログインが足りません。新しいInstagramアカウントでチェック用ログインを追加します。

LoginRequired / ChallengeRequired

使っているチェック用ログインが止まっています。別アカウントでチェック用ログインを作り、途中から再開します。

Google Sheets権限エラー

結果CSVは消さずに残します。シート共有または接続設定を直してから、件数確認から再開します。

Instagram APKが見つからない

DMGが古いか、管理者側の同梱漏れです。自分でAPKサイトを探さず、最新版DMGを受け取ります。

用語集

チェック用ログイン

詐欺チェックに使うInstagramログイン情報。ログや内部名では session / 強session と出ることがあります。

AVD / Android画面

Macの中で動くAndroid端末です。Instagramのログインはこの画面で行います。

capture

Android側のInstagram通信から、チェックに必要な情報だけを取り込む処理です。

no proxy

詐欺チェック本番では外部proxyを使わない、という意味です。初回ログイン作成時のローカル通信設定とは別です。
