1. まず結論

配布中
GitHub Release は最新あり

現在の公開最新版は 2026.07.04.3。ビルド元は 1a78a62 です。

補足
ローカル dist は参考

GitHub ActionsでReleaseを作っても、手元の dist/ は自動更新されません。

改善済み
ローカル導入も強化

Android SDK導入のタイムアウト延長と、失敗時リトライ強化を配布対象に入れます。

今の見方: 「アプリとして配布中の最新版」はGitHub Releaseと公開 latest.json を正とします。ローカルの dist/ は、そのMacで最後に作った作業用生成物です。

6. いま作業中の変更

変更 1

ops_dashboard/setup_jobs.py

Android cmdline tools導入の待ち時間を 1800秒 から 3600秒 に延長。メンバーMacでダウンロードが長い時に途中終了しにくくする変更です。

変更 2

scripts/install_android_cmdline_tools.py

SDKパッケージ導入にリトライを追加。壊れた一時ダウンロードを消して再試行するため、Error reading Zip content 系の失敗に強くなります。

位置づけ: これは「次の配布版に入れる候補」です。現在のGitHub公開最新版 2026.07.04.3 には、まだこの未コミット差分は含まれていません。

8. 注意点と次の一手

優先 やること 理由
済 READMEから固定バージョンのDMG URLを消す READMEが古いDMGを案内し続ける事故を避ける。
済 起動時に同じ版のlocalhostか確認する 古いOperatorや別ユーザーのサーバーを、最新版として開いてしまう事故を避ける。
済 Android SDK導入のタイムアウト延長とリトライ強化を入れる 初回セットアップ中の一時失敗や壊れたダウンロードキャッシュで止まりにくくする。
中 次に配る場合は make sagi-operator-release-package または GitHub Actionsを実行 pushしただけではメンバー用DMGは更新されない。
中 配布後はDMGを1回開いて初回画面まで確認 メンバー目線では、コードではなく「開ける・進める」が完了条件。
